3.Описание проекта
Название проекта: ReadAndLearn
Назначение: Онлайн-библиотека учебных материалов
Целевая аудитория:
•Пользователи (студенты и преподаватели) - ищут учебные материалы, используют фильтры, просматривают документы
•Администраторы - наполняют библиотеку, управляют категориями и типами материалов
Ключевая функциональность:
•Просмотр каталога материалов
•Фильтрация по категории, типу материал, дате добавления
•Поиск по названию с подсказками
•Детальная страница материала с описанием и просмотром
•Избранное с возможностью сохранения материалов
•Административная панель для управления материалами и категориями

6.Макеты интерфейса
Главная страница / Каталог материалов

Описание:
Главная страница представляет собой ленту учебных материалов с удобной системой навигации. Дизайн ориентирован на быстрый поиск и визуальное восприятие контента.

Верхняя панель:
Слева расположен логотип проекта «ReadAndLearn» (кликабелен, ведёт на главную).
В центре - широкая строка поиска с иконкой лупы. При вводе текста появляется выпадающий блок с подсказками (названия материалов и категорий).
Справа - иконки: «Избранное» с счётчиком количества сохранённых материалов и «Профиль» для входа/регистрации или перехода в личный кабинет.

Основная область:
Левая колонка (блок фильтров):
- Фильтр по категориям (древовидный список: например, Математика → Алгебра, Геометрия). Рядом с каждой категорией отображается количество материалов.
- Фильтр по типу материала (книги, видео, тесты, презентации).
- Фильтр по дате добавления (выпадающий список или поля «от» и «до» для выбора периода).
- Кнопка «Сбросить фильтры» для очистки всех выбранных параметров.
Правая колонка (каталог):
- Верхняя строка с сортировкой (выпадающий список: по новизне, по названию, по популярности).
- Сетка карточек материалов (3-4 колонки на компьютере; 1-3 колонки на мобильном устройстве).
- Карточка материала содержит: обложку (изображение или иконка типа файла), название материала, иконку для добавления в избранное.
- Внизу - кнопка «Загрузить ещё» для подгрузки следующих страниц.

Детальная страница материала

Описание экрана:
Страница открывается при клике на карточку материала из каталога. Дизайн разделён на логические блоки для удобного изучения информации и работы с материалом.

Верхняя часть:
- «Хлебные крошки» (навигационная цепочка): Главная / Категория / Подкатегория / Название материала. Позволяет вернуться на любой уровень.

Основная область:

Левая половина:
- Крупный предпросмотр материала: для книг - обложка. Для тестов и презентаций - иконка типа файла с возможностью скачать или открыть.

Правая половина (информационный блок):
- Название материала (крупный шрифт).
- Автор(ы) и дата публикации.
- Категория и тип материала (кликабельны - ведут на соответствующий раздел каталога).
- Краткое описание (аннотация).

Блок действий:
- Кнопка «В избранное» (добавляет материал в личный список).
- Кнопка «Читать / Смотреть» (открывает полноэкранный просмотр или ведёт на внешний ресурс).

Нижняя часть:

Вкладки с подробной информацией:
- «Описание» - полный текст аннотации или содержания.
- «Оглавление».
- «Дополнительные материалы» (ссылки, файлы для скачивания).
- «Похожие материалы» - горизонтальная карусель карточек из той же категории или с теми же тегами.

Избранное (личный кабинет пользователя)

Описание экрана:
Страница доступна авторизованным пользователям по клику на иконку «♥» в верхней панели или через профиль. Представляет собой персональную подборку сохранённых материалов.
- Заголовок: «Моё избранное» и количество материалов в списке.
- Сетка карточек: Отображение материалов в том же формате, что и в каталоге (обложка, название, автор, категория).

Действия на карточке:
- Активная кнопка «♥» для удаления из избранного.
- При клике на карточку пользователь переходит на детальную страницу материала.
- Пустое состояние: Если избранное пусто, отображается заглушка с текстом: «Здесь пока ничего нет. Добавьте материалы в избранное, чтобы не потерять их».

Админ-панель (управление материалами)

Описание экрана:
Административная панель предназначена для сотрудников, наполняющих библиотеку. Интерфейс выполнен в строгом, функциональном стиле, оптимизирован для работы с большим объёмом контента.

Верхняя панель:
- Крупная зелёная кнопка «+ Добавить материал» - открывает форму создания нового учебного материала.
- Кнопки быстрого перехода: «Категории» и «Типы материалов» - для управления справочниками.

Блок фильтрации и поиска:
- Позволяет быстро находить материалы:
- Выбор категории из выпадающего списка (с поддержкой вложенности).
- Выбор типа материала (книга, презентация, тест и т.д.).
- Поиск по названию или автору (частичное совпадение).
- Кнопка «Сбросить» для очистки всех фильтров.

Таблица материалов (колонки):
- ID (внутренний идентификатор).
- Обложка (миниатюра изображения или иконка).
- Название (кликабельно, ведёт на страницу редактирования).
- Категория и тип.
- Дата добавления.
- Автор.

Действия: иконки ✎ (редактировать), 🗑 (удалить), ↔ (посмотреть на сайте как пользователь).

Пагинация и статистика:
- Таблица разбита на страницы с возможностью выбора количества записей (25, 50, 100).
- Внизу отображается сводка: «Всего материалов: X, добавлено за неделю: Y». Это помогает отслеживать динамику наполнения библиотеки.

Админ-панель (управление категориями)

Описание экрана:
Отдельный раздел для управления иерархией категорий. Реализован в виде дерева для наглядности.

Слева - древовидный список категорий с возможностью перетаскивания (drag & drop) для изменения вложенности и порядка.

Справа - форма редактирования выбранной категории:
- Поле «Название».
- Поле «Родительская категория» (выбор из списка).
- Кнопки «Сохранить» и «Удалить» (с подтверждением, если категория не пуста).

Связь между таблицами
•categories.parent_id → categories.id (самоссылающаяся связь для поддержки вложенных категорий, например, Математика → Алгебра)
•materials.category_id → categories.id (один ко многим: одна категория может содержать много материалов)
•materials.type_id → types.id (один ко многим: один тип может быть у многих материалов)
•material_images.material_id → materials.id (один ко многим, каскадное удаление: при удалении материала удаляются и его изображения)
•table_of_contents.material_id → materials.id (один ко многим, каскадное удаление)
•additional_materials.material_id → materials.id (один ко многим, каскадное удаление)
•favorites.user_id → users.id (многие ко многим: один пользователь может иметь много избранных материалов)
•favorites.material_id → materials.id (многие ко многим: один материал может быть в избранном у многих пользователей)
